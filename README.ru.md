### [EN](./README.md)

```
Использование: ./build.sh [[<имя>][:<тег>] | [--name[=<имя>] --tag[=<тег>]]] [--deploy] [--registry=<url>]

Сборка Docker-образа с автоматическим определением параметров из Git-репозитория 
с возможностью отправки в registry.

Опции:
  <имя>[:<тег>]       Указать имя и тег образа вместе (например 'app:latest')
  --name[=<имя>]      Указать имя образа отдельно
  --tag[=<тег>]       Указать тег образа отдельно
  --deploy            Отправить образ в registry после сборки
  --registry=<url>    Явно указать URL registry (по умолчанию: автоопределение)
  --no-auto-registry  Отключить автоопределение registry
  -h, --help          Показать это сообщение

Примеры использования:
  ./build.sh                         # Автоопределение всех параметров
  ./build.sh :v1.0                   # Автоимя с указанием тега
  ./build.sh --deploy                # Сборка и отправка в автоопределенный registry
  ./build.sh --name=myapp --tag=test # Явное указание имени и тега
  ./build.sh --registry=registry.example.com --deploy  # Использование своего registry

Правила именования образов:
  1. Если имя не указано - используется путь Git-проекта (например 'группа/проект')
  2. Если registry определен - добавляется его URL
  3. По умолчанию используется тег 'latest', если не указан иной
```
